# Lark 建設業務管理パッケージ v2.0 - ユーザーマニュアル

## 概要

ISO9001準拠の建設業務管理システムです。5つのテーブルで従業員・資格・案件・工程を一元管理します。

**Base URL**: https://sjpfkixxkhe8.jp.larksuite.com/base/AiHYbDdafaAfp8slmKsjk8kKpbh

## テーブル構成

```

│  従業員マスタ   │────→│   資格記録      │
```

## 1. 従業員マスタ

### 用途
- 全従業員の基本情報を管理
- 資格記録・工程管理の参照元

### フィールド

| フィールド名 | 入力方法 | 説明 |
|------------|---------|------|
| 社員番号 | 自動採番 | E001形式で自動生成 |
| 氏名 | 手入力 | 必須 |
| フリガナ | 手入力 | 検索用 |
| 所属 | 選択 | 建築部/土木部/設備部/管理部/営業部 |
| 役職 | 選択 | 部長/課長/主任/一般 |
| 入社日 | 日付選択 | カレンダーから選択 |
| 連絡先 | 手入力 | 電話番号 |
| 状態 | 選択 | 在籍/休職/退職 |

### 入力のポイント
- **氏名**と**所属**は必須です
- 退職者は「状態」を「退職」に変更（削除しない）

## 2. 資格マスタ

### 用途
- 会社で管理する資格の定義
- 有効期限・更新周期の設定

### フィールド

| フィールド名 | 入力方法 | 説明 |
|------------|---------|------|
| 資格コード | 手入力 | Q001形式 |
| 資格名 | 手入力 | 正式名称 |
| カテゴリ | 選択 | 国家資格/民間資格/社内認定 |
| 有効期限管理 | チェック | 期限管理が必要な資格はON |
| 更新周期（年） | 数値 | 例: 5年更新なら「5」 |
| 必須部署 | 複数選択 | この資格が必須の部署 |
| 備考 | 手入力 | 補足情報 |

### 主な資格例

| 資格名 | 更新周期 | 必須部署 |
|-------|---------|---------|
| 1級建築士 | なし | 建築部 |
| 1級土木施工管理技士 | なし | 土木部 |
| 玉掛け技能講習 | なし | 建築部, 土木部 |
| 酸欠作業主任者 | なし | 設備部 |
| フルハーネス特別教育 | なし | 全部署 |

## 3. 資格記録（力量管理）

### 用途
- 従業員の資格保有状況を記録
- 有効期限アラートの対象
- ISO9001 7.2「力量」の証跡

### フィールド

| フィールド名 | 入力方法 | 説明 |
|------------|---------|------|
| 従業員 | リンク選択 | 従業員マスタから選択 |
| 資格 | リンク選択 | 資格マスタから選択 |
| 取得日 | 日付選択 | 合格日・修了日 |
| 有効期限 | 日付選択 | 期限がある場合のみ |
| 証明書番号 | 手入力 | 免許番号等 |
| 状態 | 自動/選択 | 有効/期限切れ/更新中 |
| 次回更新予定 | 自動計算 | 有効期限から自動設定 |
| 備考 | 手入力 | 補足情報 |

### 入力のポイント
- **従業員**と**資格**はリンクから選択（手入力不要）
- 有効期限があれば「状態」が自動で更新されます

## 4. 案件管理

### 用途
- 工事案件の進捗管理
- 売上・契約金額の把握

### フィールド

| フィールド名 | 入力方法 | 説明 |
|------------|---------|------|
| 案件番号 | 自動採番 | P2024-001形式 |
| 案件名 | 手入力 | 工事名称 |
| 顧客名 | 手入力 | 発注者名 |
| 現場住所 | 手入力 | 工事現場の住所 |
| 契約金額 | 数値 | 円単位 |
| 着工日 | 日付選択 | 工事開始日 |
| 竣工予定日 | 日付選択 | 完了予定日 |
| 竣工実績日 | 日付選択 | 実際の完了日 |
| 状態 | 選択 | 計画中/進行中/完了/中止 |
| 進捗率 | 自動計算 | 工程から自動計算 |
| 責任者 | リンク選択 | 従業員マスタから |
| 担当者 | 複数リンク | 従業員マスタから |
| 備考 | 手入力 | 補足情報 |

### ステータスの遷移

```
計画中 → 進行中 → 完了
           ↓
          中止
```

## 5. 工程管理

### 用途
- 案件ごとの工程（タスク）管理
- ガントチャート表示
- 進捗の実績記録

### フィールド

| フィールド名 | 入力方法 | 説明 |
|------------|---------|------|
| 案件 | リンク選択 | 案件管理から選択 |
| 工程名 | 手入力 | 作業名称 |
| 順序 | 数値 | 表示順（1, 2, 3...） |
| 開始予定日 | 日付選択 | 計画開始日 |
| 終了予定日 | 日付選択 | 計画終了日 |
| 開始実績日 | 日付選択 | 実際の開始日 |
| 終了実績日 | 日付選択 | 実際の終了日 |
| 状態 | 選択 | 未着手/進行中/完了/保留 |
| 進捗率 | 数値 | 0-100% |
| 担当者 | リンク選択 | 従業員マスタから |
| 必要資格 | 複数リンク | 資格マスタから |
| 備考 | 手入力 | 補足情報 |

### 工程状態の遷移

```
未着手 → 進行中 → 完了
           ↓
          保留
```

## 日常業務フロー

### 1. 新規案件登録

```
1. 案件管理テーブルを開く
2. 「+」で新規レコード作成
3. 案件名・顧客名・契約金額を入力
4. 着工日・竣工予定日を設定
5. 責任者・担当者をリンクから選択
6. 状態を「計画中」→「進行中」に変更
```

### 2. 工程追加

```
1. 工程管理テーブルを開く
2. 「+」で新規レコード作成
3. 案件をリンクから選択（自動入力）
4. 工程名・順序を入力
5. 予定日を設定
6. 担当者をリンクから選択
```

### 3. 進捗更新

```
1. 工程管理で該当工程を開く
2. 進捗率を更新（10%, 50%, 100%など）
3. 状態を「進行中」に変更
4. 完了時は終了実績日を入力、状態を「完了」に
```

### 4. 資格登録

```
1. 資格記録テーブルを開く
2. 「+」で新規レコード作成
3. 従業員をリンクから選択
4. 資格をリンクから選択
5. 取得日・有効期限を入力
6. 証明書番号を入力
```

## ダッシュボード・KPI

### 確認できる指標

| KPI | 説明 | 確認方法 |
|-----|------|---------|
| 進行中案件数 | 現在施工中の案件 | 案件管理でフィルタ |
| 今月完了案件 | 今月竣工した案件 | 案件管理でフィルタ |
| 期限切れ間近資格 | 30日以内に期限切れ | 資格記録でフィルタ |
| 在籍従業員数 | 現在の従業員数 | 従業員マスタでフィルタ |

### おすすめビュー

1. **ガントチャートビュー**（工程管理）
   - 工程の日程を視覚的に確認

2. **カンバンビュー**（案件管理）
   - 状態別に案件を一覧表示

3. **カレンダービュー**（資格記録）
   - 資格期限をカレンダーで確認

## よくある質問

### Q: 従業員名を間違えて入力してしまった
**A**: 従業員マスタで修正すれば、リンク先（資格記録・工程管理）は自動更新されます。

### Q: 削除したレコードを復元したい
**A**: ゴミ箱から30日以内なら復元可能です。

### Q: 進捗率が自動更新されない
**A**: 自動化スクリプトを実行してください。詳細は [自動化ガイド](./automation-guide.md) を参照。

### Q: 資格期限の通知を受け取りたい
**A**: 自動化スクリプトでLark通知を設定できます。詳細は [自動化ガイド](./automation-guide.md) を参照。

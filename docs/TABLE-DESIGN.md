# 建設業版 Lark Base テーブル設計書

## 概要

ITリテラシーが低い人でも使いやすい、シンプルで直感的なテーブル設計です。

## テーブル一覧

| No | テーブル名 | 説明 | レコード例 |
|----|----------|------|-----------|
| 1 | 工事台帳 | 全工事の契約・進捗管理 | 〇〇ビル新築工事 |
| 2 | 工程表 | ガントチャート用の工程 | 基礎工事、躯体工事 |
| 3 | 作業員マスタ | 自社・協力会社の作業員 | 田中太郎、鈴木次郎 |

## 1. 工事台帳

### フィールド一覧

| フィールド名 | 型 | 必須 | 説明 |
|------------|---|-----|------|
| 工事番号 | 自動採番 | 自動 | K-0001形式 |
| 工事名 | テキスト | 必須 | 正式な工事名称 |
| 発注者名 | テキスト | 必須 | お客様名 |
| 契約金額 | 数値 | 必須 | 円単位 |
| 契約日 | 日付 | 必須 | |
| 着工日 | 日付 | 必須 | |
| 竣工予定日 | 日付 | 必須 | |
| 実際の竣工日 | 日付 | 任意 | 完了時に入力 |
| 現場住所 | テキスト | 必須 | |
| 進捗状況 | 選択 | 必須 | 準備中/着工前/施工中/検査中/完工/引渡済 |
| 全体進捗率 | 数値 | 任意 | 0-100% |
| 関連工程 | リンク | - | →工程表（双方向） |
| 関連日報 | リンク | - | →日報（双方向） |
| 配置資機材 | リンク | - | →資機材マスタ（双方向） |
| 担当協力会社 | リンク | - | →協力会社マスタ（双方向） |
| 備考 | テキスト | 任意 | |

## 設計のポイント

### シンプルさの追求

1. **フィールド名は全て日本語**: 英語や略語は不使用
2. **選択肢は具体的**: 「A/B/C」ではなく「優良/良好/普通」
3. **必須フィールドは最小限**: 本当に必要なものだけ
4. **備考欄を必ず用意**: 迷ったときの逃げ道

### 使いやすさの追求

1. **双方向リンク**: どちらからでも参照可能
2. **自動採番**: 番号付けは自動化
3. **デフォルト値**: よく使う値を自動入力
4. **色分け**: ステータスで視覚的に判別

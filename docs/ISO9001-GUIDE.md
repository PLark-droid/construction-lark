# 建設業版Lark Base - ISO9001準拠 品質管理ガイド

> 使うだけで自然とISO9001認証が取れる、シンプルな品質管理システム

---

## このシステムでできること

**毎日の作業を記録するだけで、ISO9001に必要な品質管理記録が自動的に蓄積されます。**

| やること | 記録されること | ISO9001対応 |
|---------|--------------|------------|
| 日報を入力 | 作業記録・人員配置・進捗 | 7.1 資源管理 |
| 検査結果を入力 | 品質検査記録 | 8.6 製品の検証 |
| 写真を添付 | 施工品質の証跡 | 7.5 文書化した情報 |
| 是正を記録 | 不適合・是正記録 | 10.2 不適合への対応 |

---

## 1日5分の品質管理

### 朝（作業開始前）
1. **KY活動記録** を入力（3分）
   - 今日の作業内容
   - 危険ポイントと対策
   - 本日の安全目標

### 夕方（作業終了後）
2. **作業日報** を入力（2分）
   - 本日の進捗
   - 投入人員
   - 写真を添付

**これだけでISO9001の「記録管理」要件をクリア！**

## PDCAサイクルの回し方

### Plan（計画）
**使うテーブル**: `10_案件情報` → `11_工事契約` → `20〜22_工程管理`

1. 案件受注時に `10_案件情報` に登録
2. 契約締結時に `11_工事契約` を作成
3. 工程表を `20_大工程` → `21_中工程` → `22_小工程` で展開
4. ガントチャートビューで全体像を確認

### Do（実行）
**使うテーブル**: `40_作業日報` / `41_日報詳細` / `30_人員配置`

1. 毎朝 `51_KY活動記録` で安全確認
2. 作業終了後 `40_作業日報` で進捗報告
3. 写真は日報に添付（必須）

### Check（確認）
**使うテーブル**: `60_検査記録` / `50_安全パトロール`

1. 工程完了時に `60_検査記録` で自主検査
2. 週1回 `50_安全パトロール` で現場点検
3. ダッシュボードで進捗・品質を確認

### Act（改善）
**使うテーブル**: `52_事故災害記録` / `60_検査記録`（是正）

1. 不適合発見時 → `60_検査記録` に「要手直し」で記録
2. 是正完了後 → 同レコードに是正内容・完了日を追記
3. 重大な問題 → `52_事故災害記録` で再発防止策を記録

## 認証審査に備える

### 審査で聞かれる5つの質問と回答

**Q1: 品質目標は何ですか？**
→ `11_工事契約` を開いて「粗利率」「進捗率」を見せる

**Q2: 目標達成状況をどう管理していますか？**
→ ダッシュボードで「工事別進捗」「予実管理」を見せる

**Q3: 不適合が発生したらどうしますか？**
→ `60_検査記録` で「要手直し」→「是正内容」→「是正完了日」の流れを説明

**Q4: 協力会社はどう評価していますか？**
→ `03_協力会社マスタ` の「評価ランク」「安全成績」を見せる

**Q5: 記録はどう保管していますか？**
→ 「Lark Baseで自動保存、クラウドで永久保存」と説明

### 審査前チェックリスト

- [ ] 全工事の日報が入力されているか
- [ ] 検査記録に写真が添付されているか
- [ ] 不適合→是正の流れが記録されているか
- [ ] 安全パトロールが週1回以上実施されているか
- [ ] KY活動が毎日記録されているか

## 運用のコツ

### やってはいけないこと
- 後からまとめて日報を入力（記録の信頼性が下がる）
- 検査写真を添付しない（証拠がない）
- 是正完了日を入れない（クローズできない）

### シンプルに続けるコツ
1. **毎日同じ時間に入力**（例：17時に日報）
2. **写真は現場でそのまま添付**（後回しにしない）
3. **完璧を求めない**（80%の情報で十分）
